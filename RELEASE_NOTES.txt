NEW FEATURES & ENHANCEMENTS

1. Month-Level Budget Closure (Wife Only)
   - Added functionality to lock an entire budget month (e.g., "March 2026").
   - When a month is closed, budget allocations cannot be edited, and
     no new transactions can be attributed to that month.
   - [Web] Added a persistent lock/reopen banner on the Budget page.
   - [Mobile] Added a similar toggle banner on the Budget Screen.
   - [Add Transaction] Closed months are now disabled and marked with 🔒 
     in the budget month selector on both web and mobile platforms.

2. Post-Transaction Submission Flow
   - Replaced auto-redirect after saving a transaction with a choice:
     - "➕ Add Another" : Resets the form for a new entry.
     - "🏠 Go Home" : Returns to the dashboard.
   - [Web] Implemented as a modern success modal with motivational messages.
   - [Mobile] Implemented as a native-look modal with celebration emojis.

3. FAB Navigation Cleanup (Mobile)
   - Hidden the Floating Action Button (+) when the user is already 
     viewing the Add Transaction screen to prevent redundant navigation.
   - Standardized the 80px tab bar height across all screens.

FIXED BUGS

1. "View All" Transactions (Mobile)
   - Resolved an issue where the transaction list failed to display due 
     to an API data structure mismatch. The screen now correctly parses
     the flat transactions array from the backend.

2. Budget Save Constraints
   - Fixed a logic error where budget edits were allowed for closed months;
     now strictly enforced on the server-side.
